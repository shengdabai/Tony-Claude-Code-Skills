## Errors

### Typecheck — Exit Code 1 (recurring false-positive)

```
  File "<string>", line 1
    from claude_agent_sdk import (,AgentDefinition,ClaudeAgentOptions,ClaudeSDKClient,
                                  ^
SyntaxError: invalid syntax
  FAIL: Some template imports don't resolve
  FAIL:   Cannot import: (

=========================================
  Passed: 13
  Failed: 2
=========================================

TEMPLATE CHECK FAILED — 2 issue(s)
```

**Likely cause**: The typecheck script's import resolution check generates a malformed import string with a leading comma (`,AgentDefinition,...`) when building the import list from a template file. This is a bug in the check script itself, not in the skill templates. This same artifact has appeared in prior runs (e.g. 2026-04-04).

**Pipeline outcome**: Recorded as `success` in outcomes despite the step result being `failed` — the pipeline appears to treat this check as advisory.

**Suggested fix**: The typecheck script needs to filter out empty/null entries when constructing the import list to prevent the leading comma.
